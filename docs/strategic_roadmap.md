# 거시 로드맵

이 문서는 SpireMind의 장기 방향을 정의한다.

`development_roadmap.md`가 구현 순서를 다룬다면, 이 문서는 더 큰 질문을 다룬다. 우리는 STS2를 자동으로 깨는 도구를 먼저 만들지 않는다. 우리는 LLM이 덱빌딩 로그라이크의 전투, 성장, 위험 판단을 어디까지 처리할 수 있는지 측정한다.

## 핵심 연구 질문

> LLM은 불확실성, 누적 선택, 즉시 계산이 함께 필요한 STS2에서 어디까지 안정적으로 의사결정할 수 있는가?

이 질문은 세 부분으로 나뉜다.

1. **전투 판단**: 지금 턴에서 어떤 행동이 가장 피해를 줄이고 승리 가능성을 높이는가?
2. **성장 판단**: 지금 보상이 나중의 덱 방향에 어떤 영향을 주는가?
3. **위험 판단**: 현재 체력, 덱 상태, 지도 구조를 보고 어느 위험을 감수해야 하는가?

## 프로젝트 정체성

SpireMind는 플레이어 보조 모드가 아니다.

SpireMind는 실험 장비다. 게임 상태를 구조화하고, AI의 선택을 제한된 행동으로 실행하며, 결과를 로그로 남긴다. 플레이어는 AI를 돕는 사람이 아니라 실험을 설계하고 결과를 해석하는 사람이다.

## 장기 단계

| 단계 | 이름 | 핵심 질문 | 결과물 |
|---|---|---|---|
| S0 | 관찰 가능성 확보 | STS2 상태를 정확히 읽을 수 있는가? | 상태 JSON |
| S1 | 전투 지능 측정 | LLM은 한 턴 전투 판단을 잘하는가? | 전투 자동화 실험 |
| S2 | 성장 판단 측정 | LLM은 카드 보상과 덱 방향을 이해하는가? | 보상 선택 실험 |
| S3 | 등반 판단 측정 | LLM은 경로와 위험을 관리하는가? | 1막 등반 실험 |
| S4 | 전체 런 벤치마크 | LLM은 한 판 전체를 얼마나 잘 끝내는가? | 시드 기반 벤치마크 |
| S5 | 보정된 AI 비교 | 계산기, 규칙, 기억이 성능을 얼마나 올리는가? | 방식별 비교 리포트 |
| S6 | 학습형 정책 비교 | 강화학습 또는 imitation이 LLM보다 나은 구간은 어디인가? | 하이브리드 AI |

## S0: 관찰 가능성 확보

### 질문

게임 내부 상태를 손실 없이 읽을 수 있는가?

### 잠글 것

- 전투 상태 JSON
- 카드 더미 분리
- Power 목록
- 적 의도
- 유물
- 합법 행동 후보

### 열어둘 것

- 자동 실행
- LLM 호출
- 보상/지도/상점 선택

### 성공 기준

- 사람 눈에 보이는 전투 정보와 JSON이 일치한다.
- 사람이 직접 보기 어려운 카드 더미 정보도 정확히 들어간다.
- 상태가 바뀔 때 JSON도 늦지 않게 갱신된다.

### 결정 지점

상태 추출이 안정적이면 S1로 간다. 적 의도나 카드 더미 추출이 불안정하면 자동 플레이를 미루고 S0을 더 보강한다.

## S1: 전투 지능 측정

### 질문

LLM은 전투 상태와 합법 행동 목록만 보고 다음 행동을 고를 수 있는가?

### 잠글 것

- 행동 단위 선택
- 전투 중 LLM 호출
- 행동 실행 전 검증
- 전투 로그

### 열어둘 것

- 한 턴 전체 계획
- 보상 선택
- 장기 기억

### 성공 기준

- 일반 전투를 충돌 없이 자동 진행한다.
- LLM 응답 형식 실패가 낮다.
- 선택한 행동 대부분이 합법 행동이다.
- 사망 원인을 턴 단위로 분석할 수 있다.

### 중요한 관찰

전투에서 LLM이 실패하면 실패 원인을 분리한다.

- 계산 실수인가?
- 카드 순서 실수인가?
- 대상 선택 실수인가?
- 방어와 공격의 우선순위 실수인가?
- 적 의도를 잘못 읽은 것인가?

### 결정 지점

LLM이 계산 실수를 자주 하면 계산기를 붙인다. 전략은 괜찮지만 카드 순서가 약하면 전투 규칙 보정을 붙인다. 전투 판단 자체가 약하면 프롬프트보다 상태 표현을 먼저 점검한다.

## S2: 성장 판단 측정

### 질문

LLM은 카드 보상이 현재 덱과 미래 전투에 어떤 의미를 갖는지 판단할 수 있는가?

### 잠글 것

- 카드 보상 상태 JSON
- 현재 덱 목록
- 유물 목록
- 현재 체력과 골드
- 선택/건너뛰기 행동

### 열어둘 것

- 상점 구매
- 지도 경로
- 이벤트 선택

### 성공 기준

- 보상 선택 로그가 남는다.
- 선택 이유가 덱 방향과 연결된다.
- 단기 생존 카드와 장기 성장 카드의 선택 경향을 분석할 수 있다.

### 중요한 관찰

성장 판단은 결과가 늦게 드러난다. 그래서 한 번의 카드 선택을 바로 정답/오답으로 판정하지 않는다. 대신 이후 전투에서 그 선택이 어떤 문제를 만들었는지 태그로 연결한다.

### 결정 지점

LLM이 카드를 지나치게 많이 집으면 덱 크기 패널티를 명시한다. 강한 카드만 고르고 덱 방향을 잃으면 시너지 요약을 추가한다. 설명만 그럴듯하고 성과가 낮으면 기준선 AI와 비교한다.

## S3: 등반 판단 측정

### 질문

LLM은 지도에서 위험과 보상을 조절할 수 있는가?

### 잠글 것

- 지도 상태 JSON
- 현재 체력
- 덱 강도 요약
- 엘리트, 모닥불, 상점, 이벤트 위치
- 다음 선택 가능한 노드

### 열어둘 것

- 전체 막 장기 계획
- 보스별 맞춤 전략
- 캐릭터별 고급 경로 전략

### 성공 기준

- 1막에서 경로 선택이 자동으로 진행된다.
- 엘리트 회피와 도전의 경향을 분석할 수 있다.
- 사망 원인이 전투 판단인지 경로 판단인지 구분된다.

### 중요한 관찰

경로 판단은 STS 실력의 핵심이다. 강한 AI는 “지금 이길 수 있는 전투”보다 “다음 보스를 이기기 위해 감수할 위험”을 고른다.

### 결정 지점

LLM이 엘리트를 과하게 피하면 성장 부족으로 보스에서 죽을 수 있다. 반대로 엘리트를 과하게 고르면 체력 관리가 무너진다. 이 지점부터는 단순 승률보다 사망 패턴을 더 중요하게 본다.

## S4: 전체 런 벤치마크

### 질문

LLM은 한 판 전체를 얼마나 안정적으로 진행할 수 있는가?

### 잠글 것

- 고정 시드 묶음
- 모델 설정
- 반복 횟수
- 승리/사망 지표
- 실패 유형 태그

### 열어둘 것

- 여러 모델 비교
- 여러 캐릭터 비교
- 압축 정보 모드
- 인간 유사 정보 모드

### 성공 기준

- 같은 시드에서 반복 실험이 가능하다.
- 승률, 도달 막, 평균 사망 층을 계산할 수 있다.
- 사망 직전 로그가 자동 추출된다.
- 기준선 AI보다 어느 지점에서 나은지 설명할 수 있다.

### 결정 지점

전체 런에서 성능이 낮아도 실패가 아니다. 실패 유형이 선명하면 다음 연구 단계로 갈 수 있다. 실패 유형이 흐리면 로그 설계를 다시 고친다.

## S5: 보정된 AI 비교

### 질문

LLM 단독보다 어떤 보정이 성능을 올리는가?

### 비교할 방식

- LLM 단독
- LLM + 피해 계산기
- LLM + 카드 순서 규칙
- LLM + 위험 평가 규칙
- LLM + 덱 요약 메모리
- LLM + 기준선 정책 fallback

### 성공 기준

- 보정 방식별 성능 차이가 지표로 보인다.
- 어떤 보정이 어떤 실패 유형을 줄였는지 설명된다.
- 보정이 LLM 판단을 가리는지 확인된다.

### 결정 지점

계산기만 붙여도 성능이 크게 오르면 LLM의 약점은 산술과 즉시 계산이다. 메모리가 필요하면 장기 계획이 약점이다. 규칙 보정이 과도하게 필요하면 LLM을 전략 담당으로 제한한다.

## S6: 학습형 정책 비교

### 질문

강화학습이나 imitation policy가 LLM보다 나은 구간은 어디인가?

### 접근

- LLM 로그를 행동 데이터로 저장한다.
- 전투 행동만 학습 대상으로 제한한다.
- 보상/지도 선택은 LLM이 유지한다.
- 학습 정책과 LLM 정책을 같은 시드에서 비교한다.

### 성공 기준

- 전투 행동 정책이 LLM보다 빠르게 실행된다.
- 계산 실수와 카드 순서 실수가 줄어든다.
- 장기 선택은 LLM이 더 나은지 비교할 수 있다.

### 결정 지점

학습형 정책이 전투에서 강하면 하이브리드 구조로 간다. 학습형 정책이 특정 카드/캐릭터에만 강하면 벤치마크 대상으로 유지한다.

## 정보 공개 수준 로드맵

정보 공개는 실험 변수다.

| 수준 | 설명 | 목적 |
|---|---|---|
| 완전 정보 | 카드 더미 순서와 전체 상태 공개 | 판단 능력 상한 측정 |
| 더미 구성 정보 | 카드 더미 목록은 공개하지만 순서는 숨김 | 인간보다 많은 정보 효과를 줄임 |
| 화면 정보 | 사람이 볼 수 있는 정보만 공개 | 인간 유사 조건 측정 |
| 분석 정보 | 적 패턴, 카드 설명, 유물 설명까지 풍부하게 제공 | LLM 해석 능력 측정 |

1차는 완전 정보로 시작한다. AI가 못하면 정보 부족이 아니라 판단 문제라고 볼 수 있기 때문이다. 이후 정보 공개를 줄여서 성능 변화를 본다.

## 모델 비교 로드맵

모델 비교는 자동 플레이가 안정화된 뒤 시작한다.

1. 같은 모델, 같은 설정, 같은 시드 반복
2. 같은 모델, 다른 무작위성 설정 비교
3. 다른 모델, 같은 시드 비교
4. 빠른 모델과 강한 모델의 비용/성능 비교
5. 로컬 모델 가능성 검토

비교할 때는 승률만 보지 않는다. 응답 실패율, 평균 응답 시간, 행동 합법성, 사망 유형도 함께 본다.

## 로그 자산 로드맵

SpireMind의 핵심 자산은 로그다.

초기 로그는 사람이 읽기 쉬워야 한다. 이후에는 분석과 학습에 쓰기 쉬워야 한다.

단계:

1. JSONL 원본 로그
2. 전투별 요약 로그
3. 런별 요약 리포트
4. 실패 유형 태그
5. 모델 비교 테이블
6. 학습 데이터셋

로그가 쌓이면 다음 질문을 할 수 있다.

- LLM은 어떤 적에게 자주 죽는가?
- 방어 실패와 공격 실패 중 무엇이 많은가?
- 같은 시드에서 선택이 얼마나 흔들리는가?
- 특정 카드나 유물을 과대평가하는가?
- 장기적으로 덱을 너무 크게 만드는가?

## 공개 프로젝트 방향

퍼블릭 저장소에서는 세 가지를 분리한다.

- 실험 모드 코드
- 실험 데이터 형식
- 실험 결과 리포트

API 키, 개인 설정, 원본 게임 파일, 저작권 문제가 있는 추출물은 저장소에 넣지 않는다.

## 장르 이탈 방지

이 프로젝트는 게임을 더 쉽게 만드는 보조 모드가 아니다. 다음 방향으로 흐르지 않게 주의한다.

- 카드 추천 앱 중심 프로젝트
- 플레이어용 오버레이 중심 프로젝트
- 치트/트레이너 중심 프로젝트
- 일반 자동 클릭 도구
- STS2 데이터베이스 사이트

필요한 UI가 생겨도 목적은 실험 관찰과 분석이다.

## 최종 그림

장기적으로 SpireMind는 다음 질문에 답하는 도구가 된다.

- LLM은 STS2를 어디까지 플레이할 수 있는가?
- LLM의 실패는 계산 문제인가, 계획 문제인가, 위험 평가 문제인가?
- 게임 상태를 어떻게 표현하면 LLM 판단이 좋아지는가?
- 어떤 부분은 규칙/계산기/학습형 정책에 맡기는 편이 나은가?
- 덱빌딩 로그라이크는 AI 벤치마크로 어떤 가치가 있는가?

이 질문에 답할 수 있으면, SpireMind는 단순 자동 플레이 모드가 아니라 게임 AI 실험 플랫폼이 된다.

# 실패 처리 규칙

이 문서는 STS2 AI Runner에서 잘못된 응답, 지연, 상태 변경, 예외 상황을 처리하는 규칙을 정의한다.

실패 처리는 실험 결과를 숨기기 위한 장치가 아니다. 실패를 기록하고, 게임이 멈추지 않게 하기 위한 장치다.

## 기본 원칙

- 실패한 입력과 응답은 모두 로그에 남긴다.
- LLM 응답은 실행 전에 항상 검증한다.
- 검증 실패 시 즉시 게임 객체를 조작하지 않는다.
- 같은 상태에서 재질문은 제한된 횟수만 허용한다.
- 자동 대체 행동이 발생하면 반드시 로그에 표시한다.

## 실패 처리 표

| 상황 | 처리 | 로그 태그 |
|---|---|---|
| JSON 파싱 실패 | 같은 상태로 1회 재질문 | `invalid_response` |
| 필수 필드 누락 | 같은 상태로 1회 재질문 | `invalid_response` |
| `state_id` 불일치 | 응답 폐기 | `stale_state` |
| 없는 `action_id` 선택 | 같은 상태로 1회 재질문 | `invalid_action` |
| 현재 상태에서 더 이상 불가능한 행동 | 응답 폐기 후 새 상태 추출 | `stale_action` |
| 응답 제한 시간 초과 | 대체 행동 실행 | `timeout` |
| LLM API 오류 | 대체 행동 실행 | `api_error` |
| 행동 실행 중 예외 | 실행 중단 후 새 상태 추출 | `execution_error` |
| 게임 상태 추출 실패 | 자동 플레이 일시 중지 | `export_error` |

## 재질문 규칙

시작값:

- 같은 상태에서 재질문은 1회만 허용한다.
- 재질문에도 실패하면 대체 행동을 실행한다.

재질문 프롬프트는 원인을 짧게 알려준다.

```text
이전 응답은 실행할 수 없습니다.
이유: selected_action_id가 legal_actions에 없습니다.
같은 state_id에 대해 legal_actions 중 하나만 다시 선택하세요.
```

## 대체 행동 규칙

1차 구현의 대체 행동은 단순해야 한다.

우선순위:

1. 피해 없이 적을 처치할 수 있는 합법 행동이 있으면 실행한다.
2. 방어 카드 중 현재 에너지로 사용할 수 있는 첫 번째 행동을 실행한다.
3. 그 외에는 `end_turn`을 실행한다.

이 규칙은 강한 AI를 만들기 위한 규칙이 아니다. 실험이 멈추지 않게 하기 위한 안전장치다.

## stale_state 처리

LLM 응답이 도착했을 때 현재 `state_id`와 응답의 `state_id`가 다르면 실행하지 않는다.

예시:

```text
요청: turn_03_step_01
현재: turn_03_step_02
응답: turn_03_step_01
결과: 응답 폐기
```

이 상황은 카드 효과, 적 사망, 드로우, 외부 지연으로 발생할 수 있다.

## 실행 오류 처리

검증을 통과했지만 실행 중 예외가 발생할 수 있다.

처리 순서:

1. 예외 메시지와 스택 정보를 로그에 저장한다.
2. 같은 행동을 반복 실행하지 않는다.
3. 즉시 새 상태를 추출한다.
4. 상태 추출이 가능하면 다음 LLM 호출로 진행한다.
5. 상태 추출도 실패하면 자동 플레이를 일시 중지한다.

## 로그 필수 항목

실패가 발생하면 아래 항목을 저장한다.

```json
{
  "event_type": "llm_failure",
  "state_id": "run_ABC123_floor_06_combat_02_turn_03_step_01",
  "failure_tag": "invalid_action",
  "raw_response": "...",
  "parsed_response": null,
  "legal_action_ids": ["play_hand_0_enemy_0", "end_turn"],
  "retry_count": 1,
  "fallback_action_id": "end_turn"
}
```

## 자동 플레이 중지 조건

아래 조건에서는 자동 플레이를 멈추고 로그만 남긴다.

- 같은 전투에서 상태 추출 실패가 3회 연속 발생한다.
- 같은 전투에서 실행 오류가 3회 연속 발생한다.
- 현재 플레이어 행동 가능 여부를 판단할 수 없다.
- 협동 모드나 온라인 동기화 상태가 감지된다.

## 안전 규칙

- 싱글 플레이에서만 기본 활성화한다.
- 협동 모드에서는 기본 비활성화한다.
- 순위표나 온라인 기록과 연결된 모드에서는 사용하지 않는다.
- API 키는 로그에 남기지 않는다.
- 전체 요청 JSON에는 카드/상태 정보만 포함한다. 개인 정보나 시스템 경로는 포함하지 않는다.

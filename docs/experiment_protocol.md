# 실험 규약

이 문서는 STS2 AI Runner의 성능 측정 조건과 지표를 정의한다.

목표는 “AI가 잘했다/못했다”를 감상으로 판단하지 않는 것이다. 같은 조건에서 반복 실행하고, 실패 유형을 로그로 남긴다.

## 1차 실험 질문

> LLM은 완전한 전투 상태 JSON과 합법 행동 목록만 보고, 기본 난이도 1막 전투를 안정적으로 진행할 수 있는가?

## 고정 조건

1차 실험에서는 아래 조건을 고정한다.

- 모드: 단일 플레이
- 난이도: 승천 0
- 캐릭터: 아이언클래드
- 정보 공개: 완전 정보 모드
- 결정 방식: 행동 단위 선택
- 행동 실행: 모드 검증 후 자동 실행
- 기록 방식: 모든 요청, 응답, 실행 결과 저장

## 시작값

아래 값은 실험 시작값이다. 충분한 결과가 쌓이면 조정한다.

- 고정 시드 수: 20개
- 같은 시드 반복 횟수: 3회
- LLM 응답 제한 시간: 10초
- 잘못된 응답 재질문 횟수: 1회
- 사망 직전 보존 턴 수: 5턴

### 시작값 검증 방법

- 20개 시드에서 1막 일반 전투와 엘리트 전투가 모두 포함되는지 확인한다.
- 같은 시드 3회에서 선택 일관성이 지나치게 낮으면 모델 설정을 낮은 무작위성으로 조정한다.
- 응답 제한 시간 10초를 넘는 호출이 5%를 넘으면 프롬프트와 JSON 크기를 줄인다.
- 잘못된 응답이 5%를 넘으면 응답 형식 안내를 강화한다.

## 실험 단계

### E1: 상태 추출 검증

목표는 LLM 호출 없이 상태 JSON이 안정적으로 나오는지 확인하는 것이다.

완료 기준:

- 전투 시작부터 종료까지 상태 JSON이 끊기지 않는다.
- 카드 더미 변화가 실제 게임과 일치한다.
- 적 의도와 피해량이 화면 표시와 일치한다.
- 버프/디버프 수치가 실제 게임과 일치한다.

### E2: LLM 응답 검증

목표는 LLM이 행동을 실행하지 않고도 올바른 형식으로 응답하는지 확인하는 것이다.

완료 기준:

- 응답 JSON 파싱 성공률 95% 이상
- `state_id` 일치율 99% 이상
- `selected_action_id`가 합법 행동에 포함되는 비율 95% 이상

### E3: 전투 자동 실행

목표는 LLM 선택을 실제 전투에 적용하는 것이다.

완료 기준:

- 일반 전투 10개를 게임 충돌 없이 완료한다.
- 잘못된 응답으로 인한 강제 턴 종료가 10% 미만이다.
- 전투별 받은 피해와 사망 여부가 로그에 남는다.

### E4: 1막 등반

목표는 전투 AI가 1막 전체에서 어느 정도 버티는지 확인하는 것이다.

완료 기준:

- 고정 시드 20개에서 사망 층과 전투 로그가 모두 남는다.
- 1막 보스 도달률을 계산할 수 있다.
- 사망 직전 5턴의 판단 로그가 자동으로 추출된다.

## 주요 지표

- 승리 여부
- 도달 막
- 사망 층
- 전투별 받은 피해
- 전투별 사용 행동 수
- 턴당 평균 의사결정 시간
- JSON 파싱 실패율
- 합법 행동 선택률
- 강제 대체 행동 발생률
- 같은 시드 반복 실행 시 선택 일관성
- 사망 직전 선택 유형

## 실패 유형 태그

로그 분석 시 아래 태그를 붙인다.

- `invalid_response`: 응답 JSON이 깨짐
- `invalid_action`: 없는 행동을 선택함
- `stale_state`: 이전 상태에 대한 응답이 도착함
- `over_attack`: 방어가 필요한 턴에 공격을 과도하게 선택함
- `over_defend`: 처치 가능한 턴에 방어를 과도하게 선택함
- `bad_target`: 우선 처치 대상이 아닌 적을 공격함
- `energy_waste`: 의미 없이 에너지를 남김
- `card_order_error`: 카드 순서 때문에 손해가 발생함
- `long_term_deck_error`: 카드 보상 이후 덱 방향이 약해짐

## 기준선

LLM 성능을 해석하려면 비교 대상이 필요하다.

1차 기준선:

- 무작위 합법 행동
- 공격 우선 규칙 AI
- 방어 우선 규칙 AI

2차 기준선:

- 사람이 같은 시드 3개를 플레이한 기록
- LLM + 계산 보조
- LLM + 전투 규칙 보정

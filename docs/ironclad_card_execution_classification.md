# 아이언클래드 카드 실행 분류

## 목적

이 문서는 아이언클래드 카드 87장을 어댑터 관점에서 다시 나눈다.

분류 기준은 다음과 같다.

- 한 번 실행하고 끝나는 카드: legal action으로 카드를 사용하면 원본 `OnPlay`가 끝나고, 추가 LLM 선택이나 다음 행동 검증 전용 처리가 필요 없다.
- 후속 조작이 필요한 카드: 카드 사용 중 원본이 `CardSelectCmd`를 호출한다. LLM이 후보 중 하나를 골라야 하므로 adapter selection phase가 필요하다.
- 후속 확인이 필요한 카드: 추가 선택은 없지만 권능, X 비용, 자동 사용, 무작위, 생성 카드, 카드 내부 상태, 처치 후처리 때문에 다음 상태 또는 다음 행동 결과를 확인해야 한다.

이 분류는 카드 효과를 어댑터가 직접 재구현하기 위한 목록이 아니다. 원본 카드 실행 경로를 유지하면서, 어떤 카드에서 선택 연결과 검증을 강화해야 하는지 판단하기 위한 기준이다.

## 요약

| 분류 | 카드 수 | 어댑터 의미 |
|---|---:|---|
| 한 번 실행하고 끝 | 35 | 현재 카드 사용 legal action만으로 충분하다. |
| 후속 조작 필요 | 5 | 카드 선택 legal action을 추가로 열어야 한다. |
| 후속 확인 필요 | 47 | 원본 실행 후 상태 또는 다음 행동 결과를 검증해야 한다. |
| 합계 | 87 | `IroncladCardPool.GenerateAllCards()` 기준 전체 카드 수와 일치한다. |

|---|---|

| BloodWall | 자해 후 방어 획득으로 끝난다. |

| Breakthrough | 자해 후 전체 피해로 끝난다. |

## 후속 조작이 필요한 카드

아래 카드는 원본 카드 사용 중 플레이어 선택을 요구한다. 클릭이 아니라 legal action으로 후보를 노출해야 한다.

| 카드 | 필요한 조작 | 어댑터 상태 |
|---|---|---|
| Armaments | 비강화 상태에서 손패 카드 1장을 골라 강화한다. | `FromHandForUpgrade` 연결 및 실전 검증 완료. |
| Brand | 손패 카드 선택, 자해, 소멸, 힘 증가가 이어진다. | 선택 후보 연결 필요. |
| BurningPact | 손패 카드 1장을 골라 소멸한 뒤 드로우한다. | 선택 후보 연결 필요. |
| Headbutt | 버림 더미 카드 1장을 골라 뽑을 더미 맨 위에 올린다. | `FromSimpleGrid` 연결 필요. |
| TrueGrit | 강화 상태에서 손패 카드 1장을 골라 소멸한다. | 선택 후보 연결 필요. 기본 상태는 무작위 소멸. |

## 다음 작업 기준

1. 먼저 `Headbutt`를 대상으로 `CardSelectCmd.FromSimpleGrid` 브리지를 구현한다.
2. 같은 구조를 `BurningPact`, `TrueGrit+`, `Brand`에 확장한다.
3. 그 다음 `Stomp`, `Whirlwind`, `Cascade`처럼 비용과 자동 사용이 얽힌 카드를 실전 검증한다.
4. 마지막으로 `Unrelenting`, `Rage`, `Juggernaut`, `Unmovable`처럼 다음 행동에서 드러나는 권능 카드를 검증한다.
